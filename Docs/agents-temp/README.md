# Agents Temp Directory

## Purpose

This directory contains **temporary prompt files** generated by the `@prompt-generator` cursor command.

## What Gets Stored Here

When using `@prompt-generator` without the `--backlog` flag, the agent will:

1. Analyze your request and gather context
2. Build a detailed implementation specification
3. Save it here as `prompt-[timestamp]-[task-slug].md`

## File Naming Convention

```
prompt-YYYY-MM-DD-[task-description-slug].md
```

**Examples**:

- `prompt-2025-11-07-remove-lastname-field.md`
- `prompt-2025-11-07-add-event-filtering.md`
- `prompt-2025-11-08-refactor-auth-flow.md`

## File Lifecycle

These files are **temporary** and should be:

- ✅ Used as reference for implementation
- ✅ Deleted after task completion
- ✅ Committed if you want to track the planning process
- ⚠️ Cleaned up periodically (not part of core documentation)

## When to Use Backlog Instead

If your request is:

- Not ready for immediate implementation
- Needs prioritization discussion
- Part of long-term planning

Then use: `@prompt-generator --backlog [description]`

This will add an entry to `Docs/Backlog.md` instead of creating a file here.

## Gitignore Status

This directory is **tracked by git** but individual prompt files can be gitignored if desired.

Add to `.gitignore` if you want to keep these local-only:

```
Docs/agents-temp/*.md
!Docs/agents-temp/README.md
```
